C++ - IO

Sep 27, 2020 2020-09-27T20:55:00+08:00 by lambdaxing
7 min

  C++ 通过一族定义在标准库中的类型来处理IO。

IO 类

   <iostream> 定义了用于读写流的基本类型，<fstream> 定义了读写命名文件的类型，<sstream> 定义了读写内存 string 对象的类型。为了支持使用宽字符的语言，标准库定义了一组类型和对象来操纵 wchar_t 类型的数据。

头文件类型
iostreamistream, wistream; ostream, wostream; iostream, wiostream
fstreamifstream, wifstream; ofstream, wofstream; fstream,wfstream
sstreamistringstream, wistringstream; ostringstream, wostringstream; stringstream, wstringstream

  我们不能拷贝或对IO对象赋值，进行IO操作的函数通常以引用方式传递和返回流， 读写会改变流的状态，因此传递和返回的引用不能是 const 的。
  IO 类定义了一些函数和标志，帮组我们访问和操纵流的条件状态（condition state）。

stateexplanation
strm::iostremstrm 是一种IO类型，iostrem 是一种机器相关的类型，提供了表达条件状态的完整功能。
strm::badbit指出流已崩溃，系统级错误，不可恢复。
strm::failbit指出一个IO操作失败了，可恢复。
strm::eofbit流到达了文件结束（failbit 也会置位）。
strm::goodbit指出流未处于错误状态，此值保证为零。
s.eof()若流 s 的 eofbit 置位，则返回 true。
s.fail()若流 s 的 failbit 或 badbit 置位，则返回 true 。
s.bad()若流 s 的 badbit 置位，则返回 true 。
s.good()若流 s 处于有效状态（所有位均未置位），则返回 true 。
s.clear()将流 s 中的所有条件状态位复位，状态设置为有效，返回void。
s.clear(flags)根据给定的 flags 标志位，将 s 中的对应条件状态位复位。flags 的类型位 strm::iostate，返回void。
s.setstate(flags)根据给定的 flags 标志位，将 s 中的对应条件状态位置位。flags 的类型位 strm::iostate，返回void。
s.rdstate()返回 s 的当前条件状态，返回值类型为 strm::iostate 。

  在使用流之前检查它·是否处于良好状态，最简单的方法是将它当作一个条件来使用（等价于 !fail() ）：

1
2

    while(cin >> word)
        // ok: 读操作成功 ......

  总的来说，使用 good 或 fail 是确定流的总体状态的正确方法。表格3到6行这4个 iostate 类型的 consexpr 值表示特定的位模式，可以与位运算符一起使用来一次性检测和设置多个标志位。

1
2

    // 复位 failbit 和 badbit，保持其他标志位不变。
    cin.clear(cin.rdstate() & ~cin.failbit() & ~cin.badbit());

  每个输出流都管理着一个缓冲区，用来保存程序读写的数据。我们可以使用操作符 endl（换行）、flush（无任何额外字符）、ends（插入一个空字符）来显示刷新缓冲区。 unitbuf 告诉流接下来的每次写操作都进行一次 flush 操作，nounitbuf 重置流，使其恢复使用正常的管理系统的缓冲区刷新机制：

1
2

    cout << unitbuf;        // 所有输出操作后都会立刻刷新缓存区
    cout << nounitbuf;      // 回到正常的缓冲方式

  输入流关联到输出流时，从输入流读取数据的操作都会先刷新关联的输出流，标准库将 cout 和 cin 关联在一起。tie() 不带参数的版本返回指向关联到的那个输出流的指针，如果未关联则返回空指针。x.tie(&o) 将流 x 关联到输出流 o 。istream 对象 和 ostream 对象都可以关联到另一个 ostream 对象。每个流同时最多关联到一个流，但多个流可以同时关联到同一个 ostream 。

文件 IO

  头文件 fstream 定义了三个类型来支持文件IO: ifstream 从一个给定文件读取数据，ofstream 向一个给定文件写入数据，以及 fstream 可以读写给定文件。在创建文件流对象时，我们可以提供文件名（可选的）。提供了文件名，open 会自动调用，文件名既可以是库类型 string 对象，也可以是C风格字符数组。如果调用 open 失败或对一个已经打开的文件流调用 open，failbit 会被置位。当一个 fstream 对象被销毁时，close会自动被调用。
  每个流都有一个文件模式（file mode），用来指出如何使用文件。

文件模式含义
in以读方式打开（ifstream 和 fstream默认）
out以写方式打开（ofstream 和 fstream默认）
app每次写操作前均定位到文件末尾
ate打开文件后立即定位到文件末尾
trunc截断文件
binary以二进制方式进行IO

  默认情况下，以 out 模式打开的文件也会被截断（trunc）。保留被 ofstream 打开的文件中已有数据的唯一方法时显示指定 app或 in 模式。

string IO

  头文件 sstream 中定义的类型都继承自 iostream 头文件中定义的类型。strm.str() 返回流 strm 所保存的 string 的拷贝。strm.str() 将 string s 拷贝到流 strm 中，返回 void 。使用 istringstream 时，当 string 中的数据全部读出后，同样会触发“文件结束”信号。
